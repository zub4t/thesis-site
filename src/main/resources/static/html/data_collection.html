Data Collection

The data collection process was carried out in a laboratory at FCUP, more precisely in room 1.48, which measures 10x6x4 meters. In this room, 10 access points supporting Wi-Fi 802.11mc were placed to facilitate data collection. Specifically, Google Mesh devices were used to provide coverage throughout the room.

To collect the data, the room was divided into a grid with each square possessing an area of 1m^2. The capture was conducted in the middle of each square using a smartphone, specifically a Google Pixel 4 mounted on a tripod. This setup allowed for precise capture of the signal strength of the access points throughout the room.

The same setup was used to collect the dataset of the UWB (Ultra-Wideband) technology. However, instead of using a smartphone, a Decawave DWM1001 was used.

One hypothesis we are testing is whether using subgroups of access points can lead to better localization results than using all available APs. To test this, we are collecting data using a set of 10 access points and dividing them into subgroups of varying sizes.

80211.mc Data Set

The zip file contains 53 folders, each of which represents a stationary capture of 802.11mc data. Each folder has a unique name in the format of "EXP_X", where X is a number. Within each "EXP_X" folder, there are 10 files named "FTM_RESPONDER_0.csv" up to "FTM_RESPONDER_10.csv". These files contain information related to the Wi-Fi signal strength and performance captured during the stationary capture.

Each file contains data in the following format:

  • RSSI
  • xCoordinate
  • yCoordinate
  • zCoordinate
  • distance
  • distanceStdDevM
  • BSSID
  • SSID
  • numAttemptedMeasurements
  • numSuccessfulMeasurements
  • time
  • rangingTimestampMillis
